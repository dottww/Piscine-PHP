
<- Vector ----------------------------------------------------------------------
The Vector class allows us to represent directions of the points we want to place with our Vertex (see previous ex)

A Vector is represented by the following characteristics:
- 'x' magnitude
- 'y' magnitude
- 'z' magnitude
- 'w', homogeneous coordinate
To compare two vertices, you should normalize the W value to 1.0.

An instance of Vector can be constructed from an array of two 2 Vertex as per the following example:
$vtcXunit = new Vector( array( 'orig' => $vtxO, 'dest' => $vtxX ) );

- 'dest': the vector’s destination vertex, mandatory.
- 'orig': the vector’s origin vertex, optional, by default is worth a new instance of the x=0, y=0, z=0, w=1 vertex.

If not set, the 'orig' Vertex will default to a fresh instance with x=0, y=0, z=0, w=1

The class provides the following methods :
I. Methods with no argument:
- get_X(), get_Y(), get_Z(), get_W() : return corresponding x, y, z, w values
- Magnitude() : returns the vector’s length (or "norm") in Float datatype
- Vector normalize() : returns a normalized version of the vector with a W value of 1.0. If the vector is already normalized, returns a fresh copy of the vector
- Vector opposite() : returns the opposite vector

II. Methods with an argument of number:
- Vector scalarProduct( $k ) : returns the multiplication of the vector with a scalar number

III. Methods with an argument of right-hand side Vector:
- Vector add( Vector $rhs ) : returns the sum vector of both vectors
- Vector sub( Vector $rhs ) : returns the difference vector of both vectors
- Vector dotProduct( Vector $rhs ) : returns the scalar multiplication of both vectors in Float datatype
- Vector cos( Vector $rhs ) : returns the cosine between both vectors in Float datatype
- Vector crossProduct( Vector $rhs ) : returns the cross multiplication of both vectors (right-hand mark!)
